# AEO 测试用例库 (Test Cases)

本目录专门用于存放 AEO 系统底层检测逻辑联调测试时使用的**模拟物料与样本文件**。

这些样本直接用于验证 AI 多模态大模型 (`qwen-vl-max`) 针对特定文件格式的 OCR 提取精度，并验证系统后台对应的策略类（Handlers）能否正确执行风控阈值与拦截规则。

---

## 📂 目录结构与样本说明

### 1. 财务比率测试区 (`financial_ratio/`)
- **验证项**: 资产负债率情况
- **验证规则**: 判断连续年份内，至少存在一年的资产负债率 $\le 95\%$。
- **现有样本**: 
  - `sample_balance_sheet.png`: 模拟资产负债表，包含“负债合计”与“所有者权益合计”等关键报表字段。

### 2. 审计报告测试区 (`audit_report/`)
- **验证项**: 审计报告的合规性
- **验证规则**: 企业需要提交无保留意见的审计报告。
- **现有样本**:
  - `sample_audit_report.png`: 包含“我们审计了...”以及“无保留意见”的模拟事务所审计文件，用于验证 AI 提取“无保留意见”核心字眼的精准度。

### 3. 法务与合规测试区 (`legal_compliance/`)
- **验证项**: 遵守法律法规 (人员无犯罪记录核查)
- **验证规则**: 必须提供企业全部 4 位高管（法人、负责人、财务、关务）的无犯罪记录开具原件，人员需 100% 覆盖。
- **现有样本**:
  - `sample_compliance.png`: 模拟的派出所出具《无犯罪记录证明》，旨在指引 AI 精准抽取当事人的 `name` 和无犯罪定论。

---

## 🛠️ 如何使用这些物料？

若需要发起新的全链路实网仿真测试，请通过终端执行位于外层 `ai_test/` 目录下的仿真脚本，动态传入相应的检测项及本目录中的图片。

**参考执行动作（进入容器后）**:
```bash
# 测试《审计报告》逻辑
php /opt/www/real_sim.php "审计报告" "test_case/audit_report/sample_audit_report.png"

# 测试《资产负债率》逻辑
php /opt/www/real_sim.php "资产负债率情况" "test_case/financial_ratio/sample_balance_sheet.png"
```

> **最佳实践约定**：未来所有用于测试 OCR 边界效应、文字折损情况、含多种不同审计意见的 PDF/JPG/PNG 测试文件，都应按照业务大类建立子文件夹存放于本目录下，实现代码环境与业务物料的物理隔离。
